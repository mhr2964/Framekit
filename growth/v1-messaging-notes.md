# Framekit v1 Messaging Notes

## v1 positioning
Framekit helps creators collect feedback faster with async review rooms built for sharing work, inviting the right people, and keeping comments in one place.

## Core value prop
**Fast async creative review for creators.**
Create a review room, invite collaborators, and gather comments without chasing feedback across DMs, email, and scattered docs.

## What v1 includes
- Create a review room for a piece of creative work
- Invite collaborators into the room
- Collect comments and feedback in one place
- Run a faster async review loop without scheduling a call

## What to emphasize
- Speed to feedback
- Less chaos than fragmented review workflows
- Simple invite flow
- Clear comment collection inside a dedicated review room
- Creator-friendly language over enterprise/process jargon

## Anti-incumbent wedge
Most review workflows are patched together across messages, docs, and meetings. Framekit gives creators one async review room for invites and comments so feedback moves faster and stays organized.

## Approved claims
Use language like:
- Fast async creative review
- Review rooms for feedback
- Invite collaborators quickly
- Comments in one place
- Less back-and-forth
- Fewer scattered feedback threads
- Move from draft to decision faster

## Forbidden claims
Do **not** say or imply:
- Public share links are live in v1
- Public review pages are live in v1
- Anyone can open a link and review
- External open-link review is available now
- Share-by-link is part of the current primary CTA

Avoid phrases like:
- “Share a link with anyone”
- “Public review link”
- “Open review page”
- “No-login external review”
- “Instant link-based approvals”

## Roadmap language
If share links are mentioned, keep them clearly separate from the main message:
- “Coming next”
- “Planned for v1.1 roadmap”
- “Future share-link workflow”

Never combine roadmap language with the main CTA.

## CTA hierarchy
1. Join the waitlist
2. Request early access
3. Get early access updates

## Tone
- Concrete
- Fast
- Creator-first
- Clear over clever
- Confident, not speculative

## Reusable message rules
1. Lead with async review rooms, not abstract collaboration.
2. Mention invites and comments early.
3. Keep “faster feedback” as the main outcome.
4. Do not overpromise workflow breadth beyond rooms, invites, and comments.
5. If roadmap is mentioned, label it as future and keep it below the main CTA.
