Canonical flow v1

Create a review room, load it by room ID, and share the same link with collaborators.

This prototype is frozen to one accepted create→review→share path: start from a page URL, create a room through the versioned room route, open the room by its ID, then continue feedback through room-scoped comments.

Visible state path
  • Entry: overview routes directly into room setup
  • Create: draft, validation blocked, loading, failed, created
  • Review: loading, loaded, empty room, room not found
  • Share: copy confirmation and fallback explanation from the room URL

Accepted assumptions

Backend assumptions implied by the frozen create state

A

Room creation is versioned and synchronous for this slice

The prototype assumes POST /api/v1/rooms returns enough data to show success, a room ID, and the next review URL without extra setup steps.

B

Room loading is room-ID based

The next screen assumes GET /api/v1/rooms/:roomId loads the canonical review room and its current metadata.

C

Comments belong to the room

The review state assumes feedback is stored and queried as room-scoped comments tied to the created room rather than to a transient local draft.

State inventory

Canonical create→review→share v1 states

1

Entry point

Overview screen positions this flow as the single accepted v1 path and routes the creator directly into room setup.

2

Create states

Draft, validation blocked, create loading, creation failed, and room created. Room creation is explicitly tied to POST /api/v1/rooms.

3

Review states

Loading room, loaded room, open comments, resolved comments, no comments yet, and room not found. Room loading is explicitly tied to GET /api/v1/rooms/:roomId.

4

Share handoff

The room URL is the handoff artifact. Sharing means sending collaborators to the same room ID where room-scoped comments are visible and persistent, with copy confirmation and fallback explanation shown in-room.
