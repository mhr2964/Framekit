# Framekit Launch-Surface Decision Framework
_Date: 2026-04-29_

Use this lightweight sheet to compare the current **2 landing-page messaging directions** and their **CTA variants**, then package the recommended direction for handoff or later revision.

---

## 1) Scorecard

Score each direction/CTA pair from **1–5** on each criterion.

### Criteria definitions
- **Clarity** — A first-time visitor can quickly understand what Framekit is and why it matters.
- **Consistency with current product story** — Matches the current landing / room creation / review flow without implying features that are not present.
- **Differentiation** — Feels specific to Framekit’s calm, client-friendly website feedback angle rather than generic SaaS language.
- **Credibility** — Promise is believable based on current prototype/product state.
- **Launch readiness** — Can be used now with minimal additional interpretation or rewriting.

### Decision rule
- Choose the higher total score unless the lower-scoring option is materially clearer.
- If totals are close, prefer the option with the stronger **clarity + consistency** combination.
- Reject any option that overstates product scope even if it scores well on differentiation.

---

## 2) Approval checklist

Mark each item **Yes / No / Needs revision**.

### Clarity
- Does the headline clearly describe the product category or use case?
- Is the target user/problem obvious within one read?
- Is the CTA action explicit?

### Consistency with current product story
- Does the copy align with the current Framekit landing, create-room, and review flow?
- Does it avoid implying broader workflow, automation, or collaboration features not currently supported?
- Can the message be supported by the current prototype or accepted frontend state?

### Launch readiness
- Is the recommended direction usable now for a landing page or waitlist surface?
- Are any placeholders clearly labeled?
- Is there a single recommended headline/subhead/CTA combination, not just a set of loose options?
- Is the accompanying screenshot/social direction anchored in a real or near-final product moment?

**Approval rule:**  
Approve only if all clarity items pass and there are no unresolved consistency failures.

## 4) Recommended working use
